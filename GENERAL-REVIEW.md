## HTML 
Не пренебрегай [семантической версткой](https://doka.guide/html/semantics/). Ты должен разделять сайт на смысловые блоки, например у тебя все вполне хорошо можно разделить по `section` и добавить `main`, так не только лучше твоему браузеру ([SEO рейтинг](https://doka.guide/html/seo-for-beginners/)) и людям с disabilities ([accessibility сайта](https://doka.guide/a11y/chto-takoe-a11y/))

## JS

## В целом
Лучше добавлять .gitignore файл в репозиторий, чтобы не утекали всем такие мусорные файлы как .DS_store (маку нужны для файловой системы). Еще если бы вынес секретики в духе токенов в переменные окружения через файл .env стоило его тоже бы добавить в ignore, чтобы никто не увидел твой токен в репозитории и файл хранился бы локально. Для будущих твоих коллег можно было бы создать .env.example, где был бы пример .env файла, где не было бы секретов, а лишь примеры - вот этот файл нужно было бы залить в репозиторий, чтобы все его видели
